Web accessibility means that websites, tools, and technologies are designed and developed so that people with disabilities can use them. It ensures equal access to information and functionality for everyone, including people who are blind, deaf, have motor impairments, or cognitive disabilities. The international standard for web accessibility is the WCAG 2.1 guidelines.

This accordion uses proper ARIA attributes to communicate its state to assistive technologies:
  • aria-expanded tells screen readers whether a section is open or closed.
  • aria-controls links each button to its corresponding content panel.
  • aria-hidden hides collapsed content from the accessibility tree.
  • role="region" and aria-labelledby give each panel a descriptive name.
Screen readers like NVDA, JAWS, and VoiceOver will announce "collapsed" or "expanded" when navigating the buttons.

Full keyboard navigation is supported following the WAI-ARIA Accordion Pattern:
  • Tab / Shift+Tab — Move focus between accordion headers and other focusable elements.
  • Enter or Space — Toggle the focused accordion section open or closed.
  • Arrow Down — Move focus to the next accordion header.
  • Arrow Up — Move focus to the previous accordion header.
  • Home — Move focus to the first accordion header.
  • End — Move focus to the last accordion header.

All text and interactive elements meet WCAG 2.1 contrast requirements. The heading text (#1a1a2e on #ffffff) exceeds the 4.5:1 minimum ratio for normal text. The body text (#4b5563 on #ffffff) also meets the 4.5:1 ratio. The active indicator colour (#4f46e5) against white provides a 6.1:1 ratio, well above the 3:1 requirement for UI components. The design also respects forced-colors mode for Windows High Contrast users.
